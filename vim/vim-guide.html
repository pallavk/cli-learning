vim Field Guide.

A practical reference for modes, motions, and editing — with a quiz to test yourself.

The key mental model in vim is modes. You spend most of your time in Normal mode, where keys are commands and motions — not text. Press i, a, or o to drop into Insert mode and type; press Esc to return to Normal mode. Throughout this guide, key sequences like dw mean "press these keys in order from Normal mode," and :commands are typed in Command-line mode (reached with :).

The modes

Vim is modal. The same key does different things depending on the mode you are in.

NormalThe default. Keys are commands and motions. Return here any time with Esc.
InsertType text into the buffer. Enter with i/a/o and others.
VisualSelect text to operate on. v (characters), V (lines), Ctrl+v (block).
Command-lineEx commands and search. Reached with :, /, or ?.

Entering Insert mode

iInsert before the cursor.
aAppend after the cursor.
IInsert at the first non-blank character of the line.
AAppend at the end of the line.
oOpen a new line below and insert.
OOpen a new line above and insert.

Basic motions (Normal mode)

h j k lLeft, down, up, right.
w / b / eForward to next word start / back a word / forward to word end.
0 / ^ / $Column 1 / first non-blank char / end of line.
gg / GFirst line of file / last line of file.
3jA count repeats a motion — move down 3 lines.

Basic editing

xDelete the character under the cursor.
ddDelete (cut) the whole line.
dwDelete to the start of the next word.
cwChange a word (delete it, then Insert mode).
rxReplace one character with x.
RReplace mode — overwrite characters as you type.

Undo, redo, copy & paste

u / Ctrl+rUndo / redo.
yyYank (copy) the current line.
p / PPut after/below the cursor / before/above it.

Search & save

/textEnterSearch forward; ? searches backward.
n / NNext match / previous match.
:w / :q / :wqWrite / quit / write-and-quit. Use :q! to quit discarding changes.
Stuck in Insert mode? Press Esc. It always brings you back to Normal mode, where the rest of vim's vocabulary lives.

The operator + motion grammar

Most edits are an operator followed by a motion. Learn a few of each and they combine into hundreds of commands.

d + motionDelete over the motion. d$ deletes to line end, dG deletes to the file end.
c + motionChange (delete then insert). c$ = C.
y + motionYank over the motion. y$ yanks to line end.
dw vs dedw deletes through to the next word's start (usually eating the space); de stops at the current word's end.

Line-wise shortcuts

DDelete to end of line (= d$).
CChange to end of line (= c$).
cc / SChange the whole line (keeps indent). S is a synonym for cc.
sSubstitute one character (= cl).
JJoin the next line onto this one with a space (gJ joins without a space).

Within-line jumps

fx / FxJump onto the next/previous x on the line.
tx / TxJump just before/after x. Try dtx to delete up to it.
; / ,Repeat the last f/t search same / opposite direction.

Text objects

Operators also take text objects: i ("inner") or a ("around") plus a target. Powerful with d, c, y, v.

iw / awInner word / a word (incl. surrounding space).
ci"Change the text inside the quotes (cursor anywhere between them).
di( / da(Delete inside parentheses / including them.
di{ / ci{Delete / change the contents of the enclosing block.

Search & substitute

* / #Search forward/backward for the whole word under the cursor.
:s/old/new/Replace first match on the current line.
:s/old/new/gReplace all matches on the line (g flag).
:%s/old/new/gReplace everywhere in the file (% = all lines).
:%s/old/new/gcSame, but confirm each one (c flag).

Visual mode & indenting

v / V / Ctrl+vCharacter / line / block selection. Then d, y, c act on it.
gvReselect the previous Visual selection.
>> / <<Indent the line right / left by one shiftwidth.
gUiw / guiwUppercase / lowercase the inner word.

Marks, registers & macros

ma then `aSet mark a; jump to its exact spot. 'a jumps to the start of that line.
"ayyYank the line into named register a; put it back with "ap.
qq … qRecord a macro into register q; stop with q.
@q / 10@qReplay macro q once / ten times. @@ repeats the last macro.

Windows, buffers & tabs

:sp / :vspHorizontal / vertical split.
Ctrl+w then h/j/k/lMove focus between split windows.
:ls · :b NList buffers; switch to buffer N. :bn/:bp step next/prev.
:tabnew · gt / gTNew tab page; next / previous tab.
The dot command. . repeats your last change. Structure edits so a single . can repeat them — e.g. ciwnewwordEsc, then n to the next match and . to redo it.

Global commands (:g)

:g/pattern/cmd runs an ex command on every line matching a pattern — one of vim's most powerful tools.

:g/TODO/dDelete every line containing TODO.
:v/foo/dDelete every line that does not contain foo (:v = inverse of :g).
:g/^$/dDelete all blank lines (^$ matches an empty line).
:g/pattern/normal @qRun macro q on every matching line.

Substitution power

:%s/\s\+$//Strip trailing whitespace from every line.
& in replacementStands for the whole match. :%s/\d\+/[&]/g wraps every number in brackets.
\1 \2Back-references to \(...\) capture groups. :%s/\(\w\+\) \(\w\+\)/\2 \1/ swaps two words.
:'a,'b s/x/y/gSubstitute only between marks a and b.
&In Normal mode, repeats the last :s on the current line.

Ranges

%All lines (shorthand for 1,$).
.,$From the current line to the last line (. = current, $ = last).
10,20An explicit line range. Combine with any ex command.

Registers in depth

"0The yank register — always the last yank, untouched by deletes.
"1–"9A ring of the last nine big deletes/changes (newest in "1).
". · "% · "/Last inserted text · current filename · last search pattern.
:regShow the contents of every register.

Jumps & folds

Ctrl+o / Ctrl+iGo back / forward through the jump list. `` toggles to the pre-jump spot.
zfCreate a fold over a motion or Visual selection.
za / zo / zcToggle / open / close the fold under the cursor.
zz / zt / zbRedraw with current line centered / at top / at bottom.

Visual Block tricks

Ctrl+v → I → text → EscInsert the same text at the start of every selected line. A appends instead.
Ctrl+a / Ctrl+xIncrement / decrement the number under the cursor (counts work too).

Ex, shell & config

:r !ls -lRead a shell command's output into the buffer (:r file reads a file).
q: / q/Open the editable command-line / search history window.
~/.vimrcYour config. Reload live with :source ~/.vimrc.
:set number relativenumberHybrid line numbers: absolute on the cursor line, relative elsewhere — great for motion counts.
Pro move: combine global commands with macros. :g/pattern/normal @q applies a recorded macro to every matching line in one shot — bulk editing that would take many manual repetitions.
