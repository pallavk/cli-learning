awk Field Guide.

A practical reference for the pattern-scanning language — fields, records, and one-liners, with a quiz to test yourself.

awk processes input one record at a time (a record is normally one line). For every record it walks your pattern { action } rules: if the pattern matches, the action runs. Each record is split into fields on the field separator FS (runs of whitespace by default), reachable as $1, $2 … with $0 being the whole line.

The mental model

An awk program is a list of pattern { action } rules. For each input record, awk tests every pattern; matching rules run their action. Omit the pattern and the action runs on every line; omit the action and the default is print $0.

Fields & the whole line

$0The entire current record (the whole line).
$1, $2 …The first, second … field of the record.
$NFThe last field — NF is the field count, so $NF dereferences it.
$(NF-1)The second-to-last field.
awk '{print $1}' filePrint the first field of every line.
Whitespace is the default separator: with no -F, awk splits on runs of spaces and tabs, and leading/trailing whitespace is ignored. So " a b " has exactly two fields.

Choosing a field separator

awk -F: '{print $1}' /etc/passwdSplit on a single colon — print the username column.
awk -F, '{print $2}' data.csvSplit on commas (simple CSV without quoted commas).
awk 'BEGIN{FS="\t"} {print $3}'Set the separator to a tab inside BEGIN instead of -F.

Printing

print $1, $2Print two fields separated by OFS (a space by default).
print $1 $2No comma — the fields are concatenated with nothing between them.
print "hello", NRMix literal strings and variables; NR is the line number.
printf "%-10s %5d\n", $1, $2printf for exact formatting; you supply the \n yourself.
Comma vs no comma: print $1, $2 inserts OFS between the fields; print $1 $2 glues them together. This is the single most common beginner surprise.

Patterns: deciding which lines

/error/Run on lines where $0 matches the regex error.
$3 > 100Run when field 3, treated as a number, is greater than 100.
NR == 1Run only on the first line (e.g. to skip or print a header).
$1 == "root"Exact string comparison on the first field.

Running awk

awk '{print}' fileProgram in single quotes, file as an argument.
cat file | awk '{print $1}'awk happily reads from a pipe / stdin.
awk -f script.awk fileLoad a longer program from a file with -f.

Built-in variables

NRNumber of records read so far — the running line count across all input files.
FNRLike NR but resets to 1 at the start of each new file.
NFNumber of fields in the current record.
FSInput field separator (default: runs of whitespace).
OFSOutput field separator used by print a, b (default: a single space).
RSInput record separator (default: newline). Set to "" for paragraph mode.
ORSOutput record separator appended by print (default: newline).
FILENAMEName of the file currently being read.
OFS gotcha: changing OFS only affects a line when you rebuild a field. {$1=$1; print} forces awk to re-split and re-join $0 using the new OFS; without touching a field, print $0 comes out unchanged.

BEGIN and END blocks

BEGIN { ... }Runs once before any input is read — set FS, print headers, init counters.
END { ... }Runs once after all input — print totals, summaries.
awk '{s+=$1} END{print s}'Sum the first column, print the total at the end.
END {print NR}In END, NR holds the total number of records — a line counter.

Regex patterns

/^[0-9]+$/Whole-line match: lines that are entirely digits.
$2 ~ /^ERR/~ tests whether field 2 matches the regex.
$1 !~ /test/!~ is "does not match".
/start/,/end/A range pattern: matches every line from one matching start through the next matching end, inclusive.

Arithmetic & string functions

length($0)Length of a string; length alone means length($0).
substr($1, 2, 3)3 characters starting at position 2 (awk strings are 1-indexed).
index($0, "x")Position of the first "x", or 0 if not found.
toupper($1) / tolower($1)Case conversion.
split($0, a, ":")Split a string into array a on a separator; returns the element count.
sub(/x/, "y")Replace the first match of x in $0; returns 1 or 0.
gsub(/x/, "y", $2)Global replace in field 2; returns the number of substitutions made and edits in place.
sub/gsub mutate: they change their target ( $0 by default) in place and return a count, not the new string. Use gensub (gawk) if you want a value returned without modifying the original.

Control flow

if (x > 0) {...} else {...}Standard conditional.
for (i=1; i<=NF; i++)C-style loop, e.g. over every field.
while (cond) {...}Loop while a condition holds.
nextStop processing the current record and move to the next input line.
exitStop reading input; jumps to the END block (if any), then quits.

Associative arrays

awk arrays are always associative — subscripts are strings, even numeric-looking ones. They are perfect for counting and grouping.

count[$1]++Tally occurrences of each value of field 1.
for (k in arr) print k, arr[k]Iterate keys (order is unspecified) and print each key with its value.
if (k in arr)Test membership without creating the key (unlike arr[k], which would).
delete arr[k]Remove one element; delete arr clears the whole array (gawk).
Dedup idiom: awk '!seen[$0]++' prints each line only the first time it appears. The first time, seen[$0] is 0 (so !0 is true and it prints), then the ++ bumps it so later copies are suppressed.

User-defined functions

function max(a, b) { return a>b ? a : b }Define a function; call it like any built-in.
function f(x, tmp)Extra params after a gap are local variables — awk has no other way to scope locals.
Scoping quirk: scalars pass by value, arrays pass by reference. To get a local variable inside a function, declare it as an extra (unused-by-caller) parameter — conventionally separated by extra spaces in the parameter list.

Two files & the FNR==NR idiom

A classic awk pattern: read a lookup file fully, then process a second file using what you learned.

awk 'FNR==NR{a[$1]=1; next} ($1 in a)' f1 f2While on the first file (FNR==NR), load keys into a; on the second file, print lines whose field 1 is in a.
Why it works: NR counts across all files while FNR resets per file, so FNR==NR is only true while reading the first file. The next stops fall-through so the second rule runs only on later files.

getline

getlineRead the next record into $0 (updates NF, NR, FNR).
getline line < "file"Read one line from another file into the variable line.
"date" | getline dRun a shell command and capture its output into d.

Number formatting: OFMT & CONVFMT

OFMTFormat used when a number is printed with print (default "%.6g").
CONVFMTFormat used when a number is converted to a string in other contexts (e.g. as an array subscript). Also defaults to "%.6g".
sprintf("%.2f", x)Return a formatted string without printing — printf's value-returning sibling.

Reassigning records, RS & ORS

BEGIN{RS=""; FS="\n"}Paragraph mode: records are blank-line-separated blocks, fields are their lines.
BEGIN{ORS="\t"}Join printed records with tabs instead of newlines.
{NF=3; print}Setting NF truncates (or pads) the record's fields, rebuilding $0 with OFS.

Handy one-liners

awk '{s+=$2} END{print s}' fSum column 2.
awk '{c[$1]++} END{for(k in c) print k, c[k]}'Count occurrences per value of field 1.
awk '!seen[$0]++' fRemove duplicate lines, keeping first occurrence (no sort needed).
awk 'NF' fPrint only non-blank lines (NF is 0, i.e. false, on empty lines).
awk 'END{print NR}' fCount lines (a wc -l equivalent).
awk -F, '{print $1, $3}' OFS=, fPrint a subset of CSV columns, comma-separated.
