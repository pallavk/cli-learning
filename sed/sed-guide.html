sed Field Guide.

A practical reference for the stream editor — substitution, addresses, and the hold space — with a quiz to test yourself.

Mental model: sed reads input one line at a time into a buffer called the pattern space, runs your script against it, then by default prints the line and moves on. The -n option turns that auto-print off, so nothing appears unless you ask for it with p. Examples here assume GNU sed unless BSD/macOS is called out.

The substitute command

The workhorse of sed is s/regexp/replacement/. It finds text matching the regexp and swaps in the replacement.

sed 's/cat/dog/' fileReplace the first "cat" on each line with "dog". The file itself is not changed — the result goes to stdout.
sed 's/cat/dog/g' fileThe g flag replaces all occurrences on each line (global within a line, not across the whole file).
sed 's/cat/dog/2' fileReplace only the 2nd match on each line. Numeric flag = the Nth occurrence.
sed 's/cat/dog/gI' fileI makes the match case-insensitive (GNU); combine flags freely.
sed 's#/usr#/opt#' fileAny character can be the delimiter. Using # avoids escaping the slashes in paths.
Key fact: without g, only the first match on each line is replaced. With g, every match on that line is replaced. Neither stops sed from processing the rest of the lines.

Printing & suppressing

sed '' fileAn empty script — auto-print echoes the file unchanged (a no-op, like cat).
sed -n '4p' filePrint only line 4. -n silences auto-print, then p explicitly prints.
sed -n '$p' filePrint just the last line ($ = last line). Like tail -1.
sed '2p' fileWithout -n, line 2 prints twice — once by auto-print, once by p.

Deleting lines

sed '3d' fileDelete line 3 (just omit it from output).
sed '$d' fileDelete the last line.
sed '2,5d' fileDelete lines 2 through 5 (a comma range, inclusive).
sed '/^$/d' fileDelete blank lines (^$ matches an empty line).

Running more than one command

sed -e 's/a/b/' -e 's/c/d/' fileEach -e adds a script. They run in order on every line.
sed 's/a/b/;s/c/d/' fileA semicolon separates commands inside one script — same effect.
Memory hook: sed = stream editor. It streams a file past your edits and prints the result. It does not touch the original unless you add -i (covered in Intermediate).

Addresses: where commands apply

Put an address before a command to restrict it. No address means "every line".

sed -n '1,3p' fileLine-number range: print lines 1–3 (like head -3).
sed -n '10,$p' fileFrom line 10 to the last line (like tail -n +10).
sed '/error/d' fileRegex address: delete every line containing "error".
sed -n '/START/,/END/p' fileRegex range: print from a line matching START through the next line matching END.
sed -n '2~3p' fileGNU first~step: line 2, then every 3rd after (2, 5, 8…).
sed '3!d' file! negates the address — delete every line except line 3.
Scoping a substitution: sed '/error/s/foo/bar/' file runs the substitution only on lines containing "error". The address comes first, then the command.

Editing files in place

sed -i 's/a/b/' fileGNU: -i takes no argument and rewrites the file directly.
sed -i.bak 's/a/b/' fileGNU: attach a suffix to also save a backup as file.bak.
sed -i '' 's/a/b/' fileBSD/macOS: -i requires a suffix argument; pass an empty string for no backup.
Portability trap: GNU -i with no arg and BSD -i with an empty-string arg are incompatible. sed -i 's/a/b/' file works on Linux but treats s/a/b/ as the backup suffix on macOS.

Regex flavours: BRE vs ERE

By default sed uses basic regular expressions (BRE). -E (or GNU's -r) switches to extended (ERE).

sed 's/[0-9]\+/#/g' fileBRE: + ? ( ) { } are literal — escape them to get their special meaning (\+ = one or more).
sed -E 's/[0-9]+/#/g' fileERE: + ? ( ) { } | are special by default — no backslashes needed.
sed 's/\(ab\)\1/X/' fileBRE groups use \(...\); \1 back-references the first group.
sed -E 's/(ab)\1/X/' fileERE groups use plain (...); the back-reference is still \1.

Back-references & the whole match

sed -E 's/(\w+) (\w+)/\2 \1/' fileSwap two words: \1, \2 recall captured groups in the replacement.
sed 's/[0-9]*/(&)/' file& in the replacement = the entire matched text. Escape it as \& for a literal ampersand.

Insert, append, change & transliterate

sed '/foo/a text' filea appends a new line of text after each matching line.
sed '/foo/i text' filei inserts a new line before the match.
sed '/foo/c text' filec changes (replaces) the whole matched line(s) with new text.
sed 'y/abc/xyz/' filey transliterates char-by-char (a→x, b→y, c→z), like tr. Not a regex; sets must be equal length.

Useful s-command flags

sed -n 's/foo/bar/p' filep flag: print only the lines where the substitution succeeded.
sed 's/x/y/3g' fileNumber + g: replace from the 3rd match to the end of the line.
sed -n 's/foo/bar/w out.txt' filew file flag: write changed lines to a file.

The hold space

sed has two buffers: the pattern space (the current line) and the hold space (an extra store that persists across lines). Moving data between them unlocks multi-line transforms.

hHold: copy pattern space → hold space (overwrite).
HAppend pattern space to the hold space (with a leading newline).
gGet: copy hold space → pattern space (overwrite). Note: this is the command g, not the s///g flag.
GAppend hold space to the pattern space (with a leading newline).
xExchange: swap the pattern and hold spaces.
Reverse a file (tac): sed -n '1!G;h;$p' file. Each line appends the accumulated hold space below itself (G), saves it back (h), and the last line prints the reversed stack ($p).

Working across multiple lines

NAppend the next input line to the pattern space, joined by a \n. Now a regex can match across the two lines.
sed 'N;s/\n/ /' fileJoin each pair of lines into one (replace the embedded newline with a space).
sed ':a;N;$!ba;s/\n/,/g' fileSlurp the whole file into the pattern space, then turn every newline into a comma (joins all lines).

Branching & loops

Labels and branches give sed real control flow.

:labelDefine a label (a jump target).
b labelBranch unconditionally to the label. b with no label jumps to the end of the script.
t labelTest: branch only if an s succeeded since the last line read or last t.
Squeeze repeated spaces: sed ':a;s/ / /;ta' file. The substitution turns two spaces into one; ta loops back to :a as long as substitutions keep happening, collapsing any run of spaces down to one.

Grouping & quitting

sed '1,5{s/a/b/;s/c/d/}' fileBraces { } group commands under one address — both substitutions run only on lines 1–5.
sed '5q' fileq prints the current line, then quits. Stops at line 5, so you see lines 1–5.
sed '5Q' fileGNU Q quits immediately without printing the current line (lines 1–4).

GNU case-conversion & greediness

sed 's/.*/\U&/' fileGNU: \U uppercases the replacement (until \E); \L lowercases, \u/\l affect one char.
sed 's/.*foo//' fileGreedy: .* matches as much as possible, so this removes everything up to the last "foo". sed has no lazy *?.
sed '/file\.txt/d' fileEscape . as \. to match a literal dot — unescaped it matches any character.
Replace only the first match in the whole file: the g flag is per-line, so it cannot do this. Use the GNU range 0,/re/: sed '0,/foo/s//bar/' file ends the range at the very first match (even on line 1), unlike 1,/re/ which only starts testing the end regex from line 2.
