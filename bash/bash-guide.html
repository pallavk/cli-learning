bash Field Guide.

A practical reference for the Bourne Again Shell — variables, expansion, scripting, and more — with a quiz to test yourself.

Throughout this guide, a leading $ in examples just marks the shell prompt — you type the command after it, not the dollar sign itself. Examples assume bash 4+; a few features (associative arrays, ${var^^}, mapfile, globstar) require it.

Navigating & inspecting

The first muscle memory: where am I, what's here, and how do I move around.

pwdPrint the current working directory (absolute path).
ls -laLong listing including hidden dotfiles (-a) with details (-l).
cd dirChange directory. Bare cd goes home; cd - returns to the previous directory; cd .. goes up one level.
cat fileDump a file to the screen. Use less file to page long files, head/tail for the start/end.

File operations

touch fCreate an empty file (or update its timestamp).
mkdir -p a/b/cMake directories; -p creates parents as needed and never errors if they exist.
cp -r src dstCopy; -r for directories. mv old new renames or moves.
rm -r dirRemove. -r recurses into directories. Be careful — there is no trash.
chmod +x s.shAdd the execute bit so you can run ./s.sh.

Variables & quoting

name=valueAssign — no spaces around =. Read it back with $name (or the safer "$name").
"$var"Double quotes expand variables but keep the result as one word. Always quote unless you have a reason not to.
'literal'Single quotes are fully literal — no expansion of any kind.
export VAR=vMark a variable so child processes inherit it.
Quoting is the #1 source of bash bugs. Unquoted, $var is split on whitespace and any glob characters expand. Quoting with "$var" stops both.

Redirection & pipes

cmd > fRedirect stdout to file f, overwriting it. >> appends instead.
cmd < fFeed file f as standard input.
a | bPipe: a's stdout becomes b's stdin, e.g. ls | grep txt.
2> /dev/nullThrow away stderr (fd 2). /dev/null is the bit bucket.

Chaining commands

a ; bRun a, then b, unconditionally.
a && bRun b only if a succeeds (exit 0).
a || bRun b only if a fails (non-zero exit).
Exit status: every command sets $?. 0 means success, anything else is failure. This is what &&, ||, and if test.

Handy keystrokes

TabComplete commands and filenames; press twice to list candidates.
↑ / ↓Walk through command history.
Ctrl+rIncremental reverse search through history.
Ctrl+cInterrupt the running command (SIGINT).
Ctrl+lClear the screen (keeps your current line).
Ctrl+a / Ctrl+eJump to start / end of the line.

Globbing & brace expansion

*Any run of characters (not a leading dot). ? matches exactly one character.
[abc]A character class — any one of a, b, or c. [a-z] for ranges.
{1..5}Brace expansion generates 1 2 3 4 5. {a..e} gives letters.
cp f{,.bak}Empty alternative trick — expands to cp f f.bak.
Globs are expanded by the shell before the command runs. If nothing matches, the pattern is passed through literally by default (see nullglob in Advanced).

Command substitution

$(cmd)Run cmd and substitute its stdout, e.g. now=$(date +%F). Nests better than backticks.
$(( expr ))Arithmetic expansion: echo $(( 3 + 4 * 2 )) prints 11.

Parameter expansion

String surgery without spawning sed or cut.

${v:-def}Value of v, or def if v is unset/empty (no assignment).
${v:=def}Same, but also assigns def to v.
${#v}Length of v in characters.
${v//a/b}Replace all a with b (single slash replaces only the first).
${f%.txt}Strip shortest suffix from the end (%% = longest).
${p##*/}Strip longest leading match — effectively basename. ${p%/*} gives dirname.
${v:2:3}Substring: length 3 starting at offset 2 (zero-indexed).

Conditionals & tests

[[ ... ]]Preferred bash test keyword — safe with unquoted vars, supports &&, ||, and =~.
[[ -f p ]]-f regular file, -d directory, -e exists, -r readable.
[[ -z "$s" ]]-z empty string, -n non-empty.
(( a > b ))Arithmetic comparison. Inside [ ] use -eq -ne -lt -gt -le -ge for numbers.
[[ ]] vs [ ]: single [ ] is the old test command and is subject to word splitting — [ $x = y ] breaks if $x is empty or has spaces. [[ ]] is parsed as a keyword and avoids those traps.

Loops

for x in a b c; do ...; doneIterate over a list of words.
for ((i=0;i<10;i++)); do ...; doneC-style numeric loop with double parens.
while IFS= read -r line; do ...; done < fileThe robust way to read a file line by line.
until cond; do ...; doneLoop until the condition becomes true.

Functions

greet() { echo "hi $1"; }Define a function. Call it like a command: greet world.
$1 $2 ...Positional arguments. $# is the count, $0 the script/function name.
"$@" vs "$*""$@" = each arg as a separate word; "$*" = all args joined into one word.
local v=xScope a variable to the function so it does not leak globally.

Arrays

arr=(a b c)Indexed array, zero-based. ${arr[0]} is the first element.
"${arr[@]}"All elements as separate words. ${#arr[@]} is the count.
arr+=(d)Append an element.

History & expansion

historyList previous commands. !n reruns line n.
!!The whole previous command, e.g. sudo !!.
!$Last argument of the previous command (Alt+. inserts it interactively).

Strict mode & debugging

set -eerrexit — abort on the first uncaught failing command.
set -unounset — referencing an unset variable becomes an error (catches typos).
set -o pipefailA pipeline fails if any stage fails, not just the last.
set -euo pipefailThe common "strict mode" header for robust scripts.
set -xxtrace — print each command (prefixed with +) before running it.
errexit caveat: commands on the left of &&/||, in if conditions, or followed by || true are not aborted on, because their status is being tested. That is why cmd || true is a safe "ignore this failure" idiom.

Traps

trap 'rm -f "$tmp"' EXITRun the handler whenever the script exits (normal, error, or signal) — ideal for cleanup.
trap handler INT TERMCatch SIGINT (Ctrl-C) and SIGTERM. Note SIGKILL (9) cannot be trapped.
trap '' INTEmpty handler ignores the signal. trap - INT restores the default.

Process substitution & here-docs

diff <(cmd1) <(cmd2)<(...) exposes a command's output as a file path — no temp files.
while read x; do ...; done < <(cmd)Keeps the loop in the current shell, so variables set inside persist (unlike cmd | while).
cat <<EOF ... EOFHere-document: multi-line literal stdin. Quote the delimiter (<<'EOF') to stop expansion.
grep foo <<< "$v"Here-string: feed a single (expanded) string as stdin.
Subshell gotcha: echo x | read v leaves $v empty because the right side of a pipe runs in a subshell. Use read v <<< "x" or process substitution to keep read in the current shell.

Subshells & grouping

( cmds )Subshell — gets its own environment. cd and variable changes inside do not leak out.
{ cmds; }Group in the current shell (note the required spaces and trailing ;).
exec cmdReplace the current shell with cmd; nothing after it runs.
exec 3< fileexec with only redirections manipulates the shell's own file descriptors.

Advanced expansion

${!ref}Indirect expansion — value of the variable whose name is in ref.
${v^^} / ${v,,}Upper- / lowercase the whole value (bash 4+).
${v:?msg}Abort with msg if v is unset — a one-liner required-argument assertion.
declare -A mAssociative (string-keyed) array. ${!m[@]} are the keys, ${m[@]} the values.
mapfile -t arr < fileRead every line into an array, stripping newlines (bash 4+).

Shell options & xargs

shopt -s nullglobAn unmatched glob expands to nothing instead of the literal pattern.
shopt -s globstar** matches files recursively across directories.
find . -print0 | xargs -0 cmdBuild argument lists from stdin; NUL separators (-print0/-0) survive spaces in names.

Job control

cmd &Run in the background. $! holds its PID.
Ctrl+zSuspend the foreground job; bg resumes it in the background, fg brings it back.
jobsList background/suspended jobs of this shell.
a & b & waitRun a and b in parallel, then block until both finish.
Pro move: nohup cmd > out.log 2>&1 & detaches a job from the terminal so it survives logout. For interactive long-runners, reach for tmux instead.
